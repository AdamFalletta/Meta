Deployment Tiers

AFI BoothCast™ (Local Edge)

Pop-up multiplayer. Anywhere. No cloud required.

The **BoothCast** tier is designed for rapid deployment in temporary, high-density environments like events, conventions, or retail spaces. It is a self-contained local network server (LNS) that requires zero internet access to function, providing ultra-low latency interaction for users connected to the local Wi-Fi. It's the ultimate solution for experiential marketing and on-site activations where speed and simplicity are paramount.

Operational Example

Inside a small event venue, a BoothCast unit hosts *Knockout!* on a vertical 55” display. Attendees scan a QR code to join matches via their phones. The local server broadcasts the game state only within the building, allowing hundreds of players to compete with **zero lag**. After the event, the entire unit can be unplugged and redeployed in minutes.

Core Features

  • • **Air-Gapped Operation:** No external internet dependency.
  • • **Micro-PC Optimized:** Runs efficiently on low-power, portable hardware.
  • • **Instant Setup:** QR code-based connectivity for users.
  • • **Ultra-Low Latency:** Millisecond response times within the local network.

AFI VaultCore™ (Secure Lab)

Offline intelligence. Total control.

**VaultCore** is the ultimate environment for highly sensitive research, development, and testing where absolute data sovereignty and security are critical. It runs entirely on-premise within a physically and digitally isolated network room. By remaining completely disconnected from public networks, VaultCore eliminates external threat vectors, ensuring zero leaks and guaranteed regulatory compliance for highly confidential projects.

Operational Example

In AFI’s prototype lab, a VaultCore cluster runs early builds of *MetaOS* inside a secure network room. Access is locked down to engineers on the same subnet or VPN tunnel. Logs, backups, and build assets stay inside the VaultCore Array, under physical lock and digital key, guaranteeing top performance and **zero exposure** for firmware builds and rigorous testing.

Core Features

  • • **Air-Gapped Isolation:** Zero public internet connectivity.
  • • **Full Data Sovereignty:** Data remains under physical and digital control.
  • • **Hardened Access Control:** Strictly internal network/VPN access only.
  • • **Regulatory Compliance:** Ideal for highly regulated industry testing.

AFI MetroNet™ (Regional Public)

Regional reach. Local latency.

**MetroNet** leverages strategic co-location or partnerships with regional ISPs to offer localized public services. This tier is designed to serve users within a specific metropolitan area or country, ensuring extremely low latency for fast-paced applications like gaming or live data feeds. By keeping traffic local, we maximize performance for the end-user while minimizing our global infrastructure footprint.

Core Features

  • • **Geo-Fenced Service:** Limits accessibility to designated regions.
  • • **Latency Optimization:** Prioritizes in-region data delivery.
  • • **Local Bandwidth:** Direct access to local telecom backbones.
  • • **Public Reach:** Accessible to all users within the defined metro area.

Operational Insight

AFI partners with a local telecom to deploy MetroNet, a city-wide leaderboard hub for arcade tournaments. Hosted at the ISP’s datacenter, it’s accessible only to residents in that metro region. Users connect from home or mobile, joining brackets or downloading updates — but traffic never leaves the city, delivering **low-latency play** without the global cloud footprint.

AFI EdgeBox™ (Private Edge)

Private edge power for serious compute.

The **EdgeBox** solution is a dedicated, private compute cluster hosted at a regional colocation facility. It's geographically close to the engineering or operations teams it serves, but not accessible via the public internet. It connects securely back to the corporate network via VPN or dedicated fiber lines, providing high-performance, low-latency compute resources (like GPU rendering or ML model caching) without the latency or shared infrastructure risks of a large public cloud. The connectivity is managed through highly encrypted secure tunnels.

Key Use Case:

  • Regional AI caching and fast, local video rendering for content teams.
  • High-throughput data processing close to the point of generation.
  • Development environments requiring massive dedicated local resources.

EdgeBox Focus

Data processing and resource-intensive regional tasks.

AFI WorldHub™ (Centralized Global)

Global play, powered from home base.

**WorldHub** is AFI's solution for maintaining total, centralized control over global public-facing applications. By hosting a single, high-spec on-site server cluster directly at HQ (on-premise), AFI achieves immediate patch access, instant maintenance, and complete control over the hardware stack. While this configuration might result in higher latency for distant users, it provides unparalleled data governance and real-time operational oversight for specialized global services, such as a primary game instance or master leaderboard.

Operational Insight:

The WorldHub server powers *Space Rage* multiplayer directly from HQ. Players worldwide connect via AFI’s public IP to the main hub — a cooled, 24/7 on‑site rack with fiber uplink. It grants AFI total control over global hosting and instant patch access, embodying true **homegrown global reach**.

Scale: Global Reach

Access: Public IP/Dedicated Fiber

AFI SourceNet™ (Global Private)

A private global network for creators.

**SourceNet** serves as the internal backbone for AFI's distributed global teams. It's a privately managed, self-hosted datacenter that provides essential services like Git repositories, build systems, documentation, and asset vaults to employees worldwide through encrypted VPN tunnels. This on-premise, global reach allows AFI to maintain total data security, low costs, and high performance for internal systems regardless of the accessing user's geographic location. It provides predictable costs compared to global multi-cloud ingress/egress.

Value Proposition

  • • **Global Private Access:** Secure connectivity for international staff.
  • • **Full Data Sovereignty:** Core IP and assets never leave AFI's control.
  • • **Centralized Management:** Simplify maintenance and patching from a single hub.
  • • **Low OPEX:** Predictable costs compared to global multi-cloud ingress/egress.

AFI CloudPress™ (Public Global Cloud)

Publish to the planet.

**CloudPress** represents AFI's public-facing, globally distributed cloud infrastructure. It runs across major third-party cloud providers (AWS, Azure, GCP) and utilizes Content Delivery Networks (CDNs) to deliver content with maximum uptime and low latency to any user, anywhere in the world. This tier is designed for massive scale, auto-scaling under heavy demand, and seamless global deployment, making it ideal for web services and applications requiring universal accessibility.

Operational Insight

CloudPress runs across AWS and Azure worldwide, powering website creation and publishing for users everywhere. It orchestrates across regions, routing users to the nearest node for **lightning‑fast delivery**, ensuring global uptime and instant response regardless of peak load.

AFI CloudCore™ (Private Global Backbone)

The private backbone of the AFI ecosystem.

**CloudCore** is AFI’s invisible, highly-secure global private backbone. This network is a distributed, multi-region mesh hosted across partner datacenters worldwide. Crucially, it has no public endpoints, communicating only through highly encrypted, dedicated interlinks. It runs critical, sensitive, internal services like enterprise AI, high-stakes simulation, and inter-system data exchange—acting as the silent, secure nervous system for the entire AFI ecosystem globally.

Key Features:

  • • **Encrypted Interlinks Only:** Absolutely no public ingress/egress.
  • • **Global Distributed Mesh:** Spans multiple continents for resilience.
  • • **Enterprise AI & Simulation:** Dedicated to high-level computational tasks.
  • • **Maximum Security Posture:** Private cloud model for zero public exposure.
